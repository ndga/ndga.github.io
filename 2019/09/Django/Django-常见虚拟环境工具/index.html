Django-常见虚拟环境工具

发表于 2019-09-04 分类于 Django 阅读次数：

1. 虚拟环境的目的

  • 背景：
    要开发2个项目，一个项目是用Django1.11，另一个项目Django2.0
  • 不同的项目使用不同的环境
    • env_dj11：题库=> 项目使用Django1.11
    • env_dj20：电影=> 项目使用Django2.0
  • 保持环境整洁

2. 常见虚拟环境与管理器

  • virtualenv： 构建一个虚拟的干净的环境
  • virtualenvwrapper：基于virtualenv的一个虚拟机管理工具
  • pipenv：基于virtualenv的虚拟机管理工具
  • venv：从python3.3开始的一个虚拟环境管理工具
    • https://docs.python.org/3/library/venv.html
    • https://www.jianshu.com/p/c5f973fd34d4

3. 虚拟环境工具-venv

python3.3开始，自带的虚拟环境管理工具

  • 优点：自带工具，用起来还算方便

  • 缺点：多个虚拟环境不方便管理，需要自己维护依赖

4. 虚拟环境工具-virtualenv

比较早的一个三方开发的虚拟环境管理工具

  • 优点：用起来简单还算方便

  • 缺点：多个虚拟环境不方便管理，需要自己维护依赖

5. 虚拟环境管理器-virtualenvwrapper

基于virtualenv的一个管理环境管理工具

  • 优点：当有多个虚拟环境时，能非常方便地进行创建、删除、进入与退出

  • 缺点：需要自己管理依赖

  • 安装virtualenv

  • 安装virtualenvwrapper

  • 设置WORKON_HOME（默认路径为：家目录/Envs）

    WORKON_HOME用于管理器创建好的虚拟机保存的位置

  • virtualenvwrapper基本命令

6. 虚拟环境管理器-pipenv

基于virtualenv的一个管理环境管理工具

  • 优点：能非常方便地管理好依赖（开发、生产环境），后期部署非常方便

  • 缺点：虚拟环境与目录有绑定关系，当项目移动目录位置时，会重新创建虚拟环境
