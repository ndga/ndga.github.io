Django-项目部署

发表于 2019-08-31 更新于 2019-09-05 分类于 Django 阅读次数：

1. 常见的部署方式

  • nginx + uwsgi

    配置要求严格，效率高

  • nginx + gunicorn

    部署极其方便，用gunicorn启动Django服务的好处：

    • 能将Django包装成一个服务启动，进行管理

  • 可以起多个Django进程，提供多进程的服务，提升并发处理能力

    • supervisor监控gunicorn，一旦gunicorn进程停止服务，马上重新启动进程
  • 动静分离
    • WSGI
      处理动态请求（uwsgi, gunicorn）
    • Nginx
      处理静态资源（/static/, /uploads/）

2. Django分布式架构

  • Nginx ：
    • 提供服务，并处理静态请求
  • Server1..N：
    • 处理动态请求
  • DB：
    • 处理数据库相关
  • 静态文件：
    • 存储挂载/加Rewrite（用于放静态文件的位置）

3. 部署前的代码准备

  1. 把要安装的包写入requirements.txt文件

    • 进入虚拟环境

      Git：source venvname/Scripts/activate

      Pycharm：cd venvname/Scripts/、activate

    • 到虚拟环境文件和requirements.txt文件同级的目录下敲命令

      pip freeze > requirements.txt

  2. 收集静态文件

    • 在settings.py文件配置STATIC_ROOT

    • 收集变化(命令行敲命令)：

      python manage.py collectstatic

  3. 区分环境配置

    dev：开发环境，test：测试环境，prod：生产环境

    • 安装包
    • 数据库
    • 是否开启Debug

    • 区分环境

      • env.ini
      • 系统环境变量

    • 根据环境写配置，不同环境数据配置不同

      • 直接在settings.py中修改
      • 写settings_prod.py 、settings_test.py

    • 上传代码到GitHub，用Git工具push到远程仓库的某个分支

实验环境：Linux系统、centos7.6、python3、nginx、gunicorn、mysql
工具：一台下载了nginx软件包、能上网、可用作Server的虚拟机，关于Linux的nginx上网的配置问题可以看NAT-SNAT-DNAT

1. 安装和运行gunicorn

  • 安装

  • 在项目根目录下运行（有manage.py的那一层）

    gunicorn projectname.wsgi:application -w 4 -b hostname:port -D --access-logfile=./log/gunicorn_access.log -u username -g webadmin

    解释：

    w => worker => 进程

    b => bind => 监听（确保nginx服务器能连接上就行）

    权限 => 最小化原则, iptables 开启你需要的，默认规则Drop

  • 编写脚本run.sh

    • 服务部署脚本：安装python,安装环境
    • 停启服务脚本： service nginx stop => 写脚本
2. 配置nginx（用root用户操作）

  • 配置文件及日期文件

    • 配置文件：/etc/nginx/nginx.conf 和/etc/nginx/conf.d/default.conf
    • 日志文件：/var/log/nginx/error.log 和 /var/log/nginx/access.log

  • 配置nginx：/etc/nginx/nginx.conf

    • kill -9 + pid（进程号）

5. 常见问题

  • 安装速度慢

    • 修改pip源

  • 环境依赖

  • DB授权

  • 监听地址

  • Settings

  • Py2编码问题

  • Admin样式问题

  • 404错误（allstatic、midia、nginx.conf配置）

  • 304 Forbidden 权限错误（chmod xxx webadmin）

  • 登录页面的验证码加载不出来

  • 因为 Linux 上没有字体，将Windows里的字体文件拷贝到static/fonts/目录下

  • gunicorn不能正常启动

    • 先用 python manage.py runserver 启动
    • 看是否有错误，根据错误提示对应处理

  • nginx不能正常启动

    • 检查配置，出错了的话，检查日志 /var/log/nginx/error
